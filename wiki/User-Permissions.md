# User Permissions

Every Rekindle user is assigned a permission level that controls what they can see and do.

---

## Permission Levels

| Level | Name | Can do |
|-------|------|--------|
| 1 | Reader | Browse libraries, read archives, track reading progress |
| 2 | Downloader | Everything at level 1, plus download archives for offline use |
| 3 | Contributor | Everything at level 2, plus manually edit series metadata (About card) |
| 4 | Admin | Everything at level 3, plus manage users, libraries, API keys, uploads, and server settings |

---

## Managing Users

Admins can create, edit, and delete user accounts from **Admin Panel → Users**.

- **Create a user:** Enter a username, password, and permission level.
- **Change permission:** Tap the permission field next to any user and select the new level.
- **Reset password:** Tap the key icon next to a user.
- **Delete a user:** Tap the delete icon. This removes the account but does not delete any reading progress data from the server database.

---

## Notes

- The first account created during [First-time Setup](First-Connection-and-Setup) is always an admin (level 4).
- A user cannot elevate their own permission level, only an admin can change another user's level.
- There is no guest or anonymous access. Every user must log in with a username and password, this is to discourage piracy where possible.
